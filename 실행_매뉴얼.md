# SomaRx (InBody 처방 클리닉) 실행 매뉴얼

처음 PC를 쓰는 사람 기준으로, **이 폴더를 받은 뒤 앱이 켜질 때까지** 순서대로 적어 두었습니다.  
macOS와 Windows를 같이 표기합니다. 본인 OS만 따라가면 됩니다.

> 이 프로젝트의 가상환경 이름은 **`venv`** 입니다.

---

## 전체 한눈에 보기

1. Python 설치 확인 (없으면 설치)
2. 프로젝트 폴더로 이동
3. `venv` 가상환경 만들기
4. `venv` 활성화
5. 패키지 설치 (`requirements.txt`)
6. Tesseract OCR 설치 (필수)
7. 앱 실행 (`streamlit run app.py`)
8. 브라우저에서 사용

## 1. Python이 있는지 확인

### macOS

1. **Spotlight**(`Cmd + Space`) → `터미널` 검색 → 실행  
2. 아래를 입력하고 Enter:

```bash
python3 --version
```

- `Python 3.10.x` 또는 그 이상이면 OK → **2단계**로
- `command not found` 이면 Python 설치 필요

**Python이 없을 때 (macOS)**

1. https://www.python.org/downloads/ 접속
2. macOS용 설치 파일 다운로드 (3.10 이상)
3. 설치 후 터미널을 **완전히 종료했다가** 다시 열고 `python3 --version` 재확인

> Homebrew를 쓰는 경우: `brew install python`

### Windows

1. 시작 메뉴 → **PowerShell** 또는 **명령 프롬프트(cmd)** 실행  
2. 아래를 입력하고 Enter:

```powershell
python --version
```

또는

```powershell
py --version
```

- `Python 3.10.x` 이상이면 OK → **2단계**로
- 안 되면 Python 설치 필요

**Python이 없을 때 (Windows)**

1. https://www.python.org/downloads/ 접속
2. Windows용 설치기 실행
3. **중요:** 첫 화면에서 **“Add python.exe to PATH”** 체크
4. Install Now
5. PowerShell을 **새로 연 뒤** `python --version` 재확인

## 2. 프로젝트 폴더로 이동

프로젝트 폴더 이름은 보통 `InBody` 입니다.  
Desktop / 다운로드 / 공유 받은 위치 어디든 상관없습니다. **본인 PC의 실제 경로**로 이동하세요.

### macOS (터미널)

예: Desktop 아래 `AI Edu/InBody`에 둔 경우

```bash
cd "/Users/여기본인계정/Desktop/AI Edu/InBody"
```

경로에 빈칸이 있으면 **따옴표**로 감싸야 합니다.

폴더가 어디 있는지 모르겠다면:

1. Finder에서 `InBody` 폴더 열기
2. 경로 표시줄에서 폴더를 터미널로 드래그하거나
3. Finder에서 폴더 우클릭 → 서비스를 쓰거나, 터미널에서 `cd ` 입력 후 폴더를 드래그앤드롭

확인:

```bash
ls
```

`app.py`, `requirements.txt`, `parser.py` 가 보이면 올바른 폴더입니다.

### Windows (PowerShell)

예: Desktop에 `InBody`를 둔 경우

```powershell
cd "$env:USERPROFILE\Desktop\InBody"
```

다른 위치면 본인 경로로 바꿉니다. 경로에 빈칸이 있으면 따옴표를 사용합니다.

```powershell
cd "C:\Users\홍길동\Desktop\AI Edu\InBody"
```

확인:

```powershell
dir
```

`app.py`, `requirements.txt` 가 보이면 OK.

## 3. 가상환경 `venv` 만들기

프로젝트 폴더 안에서 실행합니다.  
이미 `venv` 폴더가 있다면 **3-1**을 보세요.

### macOS

```bash
python3 -m venv venv
```

### Windows

```powershell
python -m venv venv
```

(`python`이 안 되면 `py -m venv venv`)

성공하면 프로젝트 안에 `venv` 폴더가 생깁니다.

### 3-1. 이미 `venv`가 있는 경우

- **본인이 예전에 이 PC에서 만든 `venv`** → 만들 필요 없이 **4단계(활성화)**로
- **다른 사람 Mac/Windows에서 복사해 온 `venv`** → 삭제 후 다시 만들기

```bash
# macOS / 공용
rm -rf venv
python3 -m venv venv
```

```powershell
# Windows
Remove-Item -Recurse -Force venv
python -m venv venv
```

## 4. 가상환경 활성화 (매번 실행 전에 필요)

앱을 켤 때마다 **활성화된 상태**여야 합니다.  
프롬프트 앞에 `(venv)` 가 보이면 성공입니다.

### macOS

```bash
source venv/bin/activate
```

### Windows (PowerShell)

```powershell
.\venv\Scripts\Activate.ps1
```

**실행 정책 오류가 나면 (Windows)**  
한 번만 아래를 실행한 뒤, 다시 활성화를 시도합니다.

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### Windows (명령 프롬프트 cmd)

```cmd
venv\Scripts\activate.bat
```

비활성화(종료할 때, 선택):

```bash
deactivate
```

## 6. Tesseract OCR 설치 (필수 · OS마다 한 번)

이 앱은 PDF를 글자로 읽기 위해 **Tesseract**가 필요합니다.  
Python 패키지 설치와 **별개**입니다.

### macOS

터미널에서:

```bash
brew install tesseract tesseract-lang
```

Homebrew가 없다면 https://brew.sh 안내대로 설치한 뒤 위 명령을 실행합니다.

확인:

```bash
tesseract --version
tesseract --list-langs
```

목록에 **`kor`**, **`eng`** 가 있어야 합니다.

### Windows

1. 브라우저에서 열기:  
   https://github.com/UB-Mannheim/tesseract/wiki
2. Windows용 설치 파일(`.exe`) 다운로드
3. 설치 실행
4. 설치 중 **Additional language data** / 언어 선택에서  
   - **Korean**  
   - **English**  
   반드시 선택
5. 가능하면 **Add to PATH** 옵션을 켭니다  
6. 기본 설치 경로 예:  
   `C:\Program Files\Tesseract-OCR\`

설치 후 **PowerShell을 새로 열고** 확인:

```powershell
tesseract --version
tesseract --list-langs
```

`kor`, `eng`가 보이면 OK.

#### PATH에 안 잡힐 때 (Windows)

1. 시작 메뉴 → “환경 변수” 검색 → **시스템 환경 변수 편집**
2. 사용자 변수에 새로 만들기  
   - 이름: `TESSERACT_CMD`  
   - 값: `C:\Program Files\Tesseract-OCR\tesseract.exe`  
     (실제 설치 경로에 맞게)
3. PowerShell을 완전히 종료했다가 다시 실행

또는 설치 폴더의 `tesseract.exe` 경로를 PATH에 추가해도 됩니다.

### Linux (참고)

```bash
sudo apt install tesseract-ocr tesseract-ocr-kor tesseract-ocr-eng
```

## 7. 앱 실행

프로젝트 폴더 + `(venv)` 활성화 상태에서:

```bash
streamlit run app.py
```

성공하면

- 터미널에 주소가 뜹니다 (예: `http://localhost:8501`)
- 브라우저가 자동으로 열리기도 합니다

안 열리면 브라우저 주소창에 `http://localhost:8501` 을 직접 입력합니다.

### 종료

터미널에서 `Ctrl + C` (Mac도 동일)

## 자주 생기는 문제

### Q1. `streamlit: command not found` / 인식 안 됨

- `(venv)` 가 앞에 붙어 있는지 확인
- 활성화 후 다시: `pip install -r requirements.txt`

### Q2. 앱이 “Tesseract를 찾을 수 없습니다”라고 함

- 6단계 설치·언어팩 확인
- Windows는 `TESSERACT_CMD` 설정 후 터미널 재실행
- `tesseract --list-langs`에 `kor`, `eng` 있는지 확인

### Q3. PDF는 올렸는데 숫자가 거의 비어 있음

- OCR 실패 또는 다른 양식의 결과지일 수 있음
- **원본** 탭에서 이미지·OCR 텍스트 확인
- 사이드바에서 주요 지표 수동 입력

### Q4. Windows에서 `Activate.ps1` 실행이 막힘

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

그다음 다시 `.\venv\Scripts\Activate.ps1`

### Q5. 포트가 이미 사용 중

```bash
streamlit run app.py --server.port 8502
```

### Q6. 글꼴이 조금 다르게 보임

- 인터넷이 되면 Google Fonts로 동일하게 보입니다
- 오프라인이면 Windows는 맑은 고딕 등으로 대체됩니다 (정상 동작)

### Q7. Mac의 `venv`를 Windows로 복사했더니 깨짐

- `venv`는 **OS마다 새로** 만드세요 (3단계)
